程序逻辑:
    xml解析和ocr识别关键字比对, 结合登录框ui布局的特点.
    先依据布局特点判断是登录框的可能性, 再分别找出各控件的坐标
    1.用户登录框和密码框class类型为edittext, 先解析xml后放到list中,依据关键字筛选和控件布局特点
    2.如果用户框和密码框依据xml解析得不到或者有多个,再进行ocr关键字识别.但如果edittext已经有登录用户名或密码信息(依据正则查询账户规则),也需要做处理
    3.识别出用户框和密码框,再分别以xml和ocr方式识别登录按钮(button/textview), 多个时依据class,坐标和关键字是否完全匹配进行筛选

异常情况:

    1.截图和XML以url文件提供
    2.接口输出登录按钮, 用户名, 密码框坐标
    3.统一发布时采用http服务,目前要求python以命令行方式传入和输出数据
    4.最好统一为utf8编码.关键字配置数据直接写到代码中时, 程序OK, 但以配置数据读取时中文字符串匹配不成功~!~,跟中文字符编码有关系
    5.登录按钮多样,可能是button, 也可能是textview,
    6.用户名和密码框除了edittext, 也可能是android.view.View, 有Ui的登录按钮必须是 用户名和密码框有输入属性才能enable, 不能依据clickble属性判断, 依据坐标判断 text和resource_id可能都为空,需结合ocr
    7.验证是否是登录框,用户或密码框先不要点击,会有黑色光标,影响ocr识别
    8.逻辑中, 在用户和密码框下的登录框候选集可能有多个
    9.用户和密码框类型可能是editext之外的类型吗? 如果是,有可能会得到多个候选项
    10.按钮如果是圆角,图像二值化处理时会有黑边或白边, 会导致ocr识别失败,但不能随便裁剪边框,会裁剪到字
    11.登录按钮不一定是android.widget.Button和android.widget.TextView, 其他的android.view.View都有可能.
    12.某些app(Ersi)针对不同的手机, xml内容动态变化, 完全无法获取xml中的控件信息, 如果截屏中没有明显信息通过ocr也识别不到, 无法准确获取对话框内容.
